# 安全治理与生产化基线

## 适用边界

当前仓库是竞赛原型，不应直接处理个人信息、未发布商品、供应商报价、合同、门店明细或任何生产凭据。本文给出试点前控制基线，不代表已通过 ISO、等保、隐私或行业合规认证。

## 核心威胁

| 威胁 | 典型场景 | 最低控制 |
|---|---|---|
| 提示注入 | 社媒文本夹带“忽略规则并泄露数据” | 输入视为不可信数据；系统指令隔离；工具白名单；输出验证 |
| 敏感数据泄露 | Prompt、日志或向量库包含未发布商品/个人信息 | 数据分级、最小化、脱敏、加密、保留期和访问审计 |
| 过度代理 | Agent 自动打样、采购、定价或发布 | 高影响工具默认禁用；人工审批；额度和品类边界；双人复核 |
| 供应链与模型风险 | 依赖或权重被替换 | 锁定版本、制品签名、SBOM、漏洞扫描、来源和许可证记录 |
| 错误建议与自动化偏见 | 虚拟面板把单一模型偏见放大 | 外部效度测试、分群误差、拒答/降级、置信区间、人工最终决策 |
| IP 与内容侵权 | 生成内容模仿品牌/IP 或训练材料权利不清 | 许可清单、相似度复核、法务/品牌审批、生成记录和素材血缘 |
| 评估污染 | 未来销量列、同款 SKU 或数据延迟进入训练 | 特征可用时间、隔离窗、SKU 族切分、盲测集封存、逐窗审计 |

## 生产架构原则

1. **身份优先**：人、服务和 Agent 使用独立身份；不共享长期密钥；按任务授予最小权限。
2. **数据平面隔离**：原始数据、特征、Prompt/响应、偏好记录和评估结果分区存储；分别定义保留期。
3. **模型网关**：统一模型调用、超时、重试、配额、内容策略、版本和成本记录；禁止业务代码直连未知模型端点。
4. **工具代理**：读工具与写工具分离；写操作要求策略检查和审批令牌；参数做结构化校验。
5. **可观测性**：记录 trace_id、数据快照、Prompt 模板版本、模型版本、工具调用、延迟、错误、审批和最终动作。
6. **故障安全**：模型不可用、区间过宽、数据漂移、策略冲突时降级为“不建议”，而不是自动给出确定性结论。

## 上线门禁

- 安全：威胁模型完成；高危问题清零；红队覆盖提示注入、数据外带和越权工具。
- 数据：字段分级和可用时间明确；删除与保留流程可执行；真实标签定义获业务确认。
- 模型：离线盲测、分群表现、校准、稳定性和漂移阈值通过；有旧版本回滚包。
- 业务：产品经理能查看证据和不确定性；否决不会被绕过；试点影响面有限。
- 法务：个人信息、数据跨境、第三方内容、模型/数据许可证和 IP 授权完成审查。
- 运营：事件响应、值班、变更、模型卡、数据卡和季度复评机制明确。
