# 现场评审演示导览（6 分钟）

本页供赛事评委与赛题企业代表提前了解现场演示的信息顺序和证据位置。演示以业务问题、可核验证据和企业验证路径为主，不把开发过程或团队内部安排作为评审内容。

## 0:00-0:40 企业问题

MINISO 2025 年年报披露，MINISO 品牌平均每月推出约 1,600 个 SKU。高密度上新场景的核心挑战不是创意数量不足，而是如何更早识别有效趋势、筛选可制造概念，并用可追溯证据决定哪些候选值得进入打样与试点。

**评审结论：** 本方案定位为新品研发决策支持系统，不是批量生成创意的工具。

## 0:40-1:30 方案架构

现场将呈现“趋势验证 → 结构化创意 → 进化与 DFM → 校准面板 → 决策工作台 → 人工审批”的闭环，以及贯穿全链的时光机回测与治理平面。

**评审结论：** 基座模型可以替换；时间化数据、评估协议、制造规则、偏好记录和审计血缘才是可积累的企业资产。

## 1:30-3:30 可复现证据

技术复核命令如下，业务评审无需现场操作：

```bash
pytest -q
python scripts/run_backtest.py
```

现场证据包括自动化测试、回测图表和 `metrics.json` 的逐窗口记录。评审可核对训练期后的 14 天隔离窗，以及候选评分侧不可见的 `is_hit`、`sales_90d`、`realized_margin` 字段。

**证据边界：** 所有数字来自合成数据，只验证管线与实验协议，不代表名创优品真实 KPI。

## 3:30-4:40 两项关键技术设计

1. **校准虚拟面板：** 以真实观察训练锚定映射，以独立留出数据计算保形残差并输出区间；缺少真实锚点时不报告“购买概率”。
2. **进化与 DFM：** 同时评估需求潜力、可制造性与新颖性，返回非支配前沿，由产品经理选择权衡点。

## 4:40-5:30 企业联合验证路径

建议依次执行历史盲测、影子运行和受控试点。采购、打样、定价与上架均保留人工审批、理由、审计和回滚。

**评审结论：** 竞赛验证版不直接接入生产；企业真实数据和受控试点决定其业务价值。

## 5:30-6:00 总结

本项目没有以合成数据证明商业提升，而是先提交一套可复现、可校准、可治理的技术基线。若进入企业联合验证，首要工作是共同冻结标签、业务基线、盲测期和验收阈值，再由真实数据决定是否进入影子运行。

## 常见评审问题

- **为什么 Lift 不是固定大于 1？** 随机排序的单窗口结果会波动，Lift 的随机期望才约为 1；报告保留平均值与全部窗口，不删除不利结果。
- **Granger 是否代表因果？** 它只表示所设模型与样本内的预测领先性，不等于结构因果；还需反向检验、多重比较修正和业务事件分析。
- **Agent 失败时如何处理？** 语言任务暂停或降级；确定性回测、DFM 和校准仍可独立运行；生产环境不允许用静默模拟结果替代真实模型输出。
