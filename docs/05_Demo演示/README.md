# 现场 Demo 演示脚本（6 分钟）

## 演示目标

让评委看到三件事：项目能复跑、不会偷看未来、知道当前证据边界。

## 0:00-0:40 业务问题

“名创优品 2025 年年报披露 MINISO 品牌平均每月上新约 1,600 个 SKU。我们解决的不是创意数量，而是如何在高密度决策中更早发现趋势、筛选可制造概念，并用证据决定哪些值得进入打样。”

## 0:40-1:30 架构

展示 README 的架构图：趋势 → 创意 → 进化/DFM → 校准面板 → 决策工作台；回测和治理贯穿全链。

关键话术：“模型可以替换，时间数据、评估协议、规则和偏好记录才是企业资产。”

## 1:30-3:30 可复现回测

```bash
pytest -q
python scripts/run_backtest.py
```

展示测试通过、PNG 和 `metrics.json` 的首个窗口。指出：训练期结束后有 14 天隔离；候选侧看不到 `is_hit`、`sales_90d`、`realized_margin`。

展示结果前先说：“以下数字来自合成数据，只验证管线，不代表名创真实 KPI。”

## 3:30-4:40 两个技术细节

1. 虚拟面板：训练锚定映射，留出数据计算保形残差，输出区间；没有真实锚点时不报“购买概率”。
2. 进化搜索：需求、DFM、新颖性三个目标，返回非支配前沿；产品经理选择权衡点。

## 4:40-5:30 企业落地

历史盲测 → 影子运行 → 受控试点。采购、打样、定价、上架全部保留人工审批、理由和回滚。

## 5:30-6:00 收尾

“我们没有用合成数据证明商业 uplift；我们先证明系统经得起复现和追问。下一步与企业共同冻结标签、基线和盲测期，让真实数据决定是否进入影子运行。”

## 备用问答

- **为什么 Lift 不是固定大于 1？** 随机排序单窗会波动，Lift 的随机期望才约为 1；报告平均和逐窗，不删不利窗口。
- **Granger 是因果吗？** 是模型内的预测领先性，不是结构因果；还要做反向检验、多重比较修正和业务事件分析。
- **Agent 失败怎么办？** 语言任务暂停或降级；确定性回测、DFM、校准仍可独立运行；生产中不允许静默模拟。
