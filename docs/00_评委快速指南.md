# 评委快速指南：3 分钟读懂项目

> **团队、服务对象与边界：** 本项目由三名学生组队完成，服务对象是需要改善新品研发决策的企业。团队提供面向企业真实工作流的竞赛验证版解决方案；它不是企业内部项目，也不代表已经完成生产部署。

## 一句话结论

这不是一个“让大模型批量想创意”的工具，而是一套帮助企业把**趋势发现、创意筛选、上市前验证、产品经理审批和结果复盘**连成闭环，并且能在历史时点上被严格回测的决策系统。

## 评审时请先区分三种证据

| 标签 | 含义 | 本仓库对应内容 |
|---|---|---|
| 已实现 | 代码存在、可运行、有自动化测试 | 回测切片、指标、校准、DFM、帕累托前沿、Agent 编排 |
| 合成演示 | 用固定生成器验证端到端逻辑 | `artifacts/demo/` 的 JSON、CSV、PNG |
| 企业待验证 | 必须拿真实历史数据或试点才能回答 | 爆品命中率提升、周期缩短、打样节省、ROI |

项目刻意不把第二类写成第三类。当前合成实验的作用是证明：系统能复跑、窗口不穿越、未来结果列不会泄漏、指标计算一致。

## 最有区分度的五点

1. **可回测**：滚动窗口不是口号。代码按 180 天训练、14 天隔离、90 天预测、30 天步长执行，并在 JSON 中保留每窗审计信息。
2. **敢表达不确定性**：虚拟消费者只作为弱信号；评分要用真实观察锚定，并以保形区间表达风险。
3. **真多目标**：进化模块保留需求潜力、可制造性、新颖性三个目标，返回非支配解，而非给单一加权分改名“帕累托”。
4. **信号要验证领先性**：趋势热度先通过滞后方向和 Granger 检验，再进入决策，不默认“热度高就是未来会卖”。
5. **人在环中**：AI 输出是候选和证据包，不直接触发采购、定价、打样或上架。

## 5 分钟现场复现

```bash
python -m venv .venv
.venv\Scripts\python -m pip install -r requirements-dev.txt
.venv\Scripts\python -m pytest -q
.venv\Scripts\python scripts\run_backtest.py
```

如果使用 macOS/Linux，把 `.venv\Scripts\python` 换成 `.venv/bin/python`。

预期看到 15 项测试通过，并在 `artifacts/demo/` 生成三类证据。固定种子下，表格模型的合成结果为 Precision@20 0.490、Lift@20 1.752、NDCG@20 0.471。请注意，这不是名创真实 KPI。

## 建议追问

**“怎么证明没有未来信息？”**

预测器收到候选集前，`is_hit`、`sales_90d`、`realized_margin` 被引擎删除；测试会主动断言这些列不可见。每个窗口的受保护列和日期边界写入 `metrics.json`。

**“虚拟消费者靠谱吗？”**

不能默认靠谱。系统把它定位为待校准的弱测量：训练锚定映射、留出校准残差、输出覆盖区间，并要求与真实调研、评论行为和销量做外部效度对照。

**“多 Agent 是不是包装？”**

Agent 只承担不同输入/输出契约的任务；真正壁垒是时间化数据、评估协议、DFM 规则、偏好记录和治理。模型可以替换，证据链保留。

**“如何进入企业？”**

先做历史盲测，再影子运行，最后受控试点。任何写入生产系统的动作都需要人工审批、审计和回滚。
