# 评审演示工作台

本目录是面向赛事评委与赛题企业代表的静态演示前端。页面不依赖后端或在线模型，直接展示公开用户反馈洞察、仓库中已实现的决策流程、合成回测证据和企业验证边界。

## 本地查看

直接打开 `index.html`，或在仓库根目录运行：

```bash
python -m http.server 4173 --directory site
```

然后访问 `http://127.0.0.1:4173/`。

## 演示路径

1. 用户洞察：浏览公开反馈主题，并查看它如何转译为产品动作与决策字段；
2. 趋势证据：对比通过、观察和不进入三类信号；
3. 创意前沿：筛选非支配候选并进行双候选权衡；
4. 上市前验证：查看校准弱信号和分群区间；
5. 人工决策：记录进入盲测、补充证据或本轮驳回；
6. 回测与治理：核对证据等级、协议参数和逐窗口结果。

## 证据口径

- `assets/metrics.json`、`assets/backtest_summary.csv` 和 `assets/backtest_metrics.png` 来自仓库版本化合成回测产物；
- 用户洞察聚合自 `docs/00_用户真实反馈洞察/` 的 124 条公开样本，其中 119 条进入分析；该数据为目的性采样，不代表消费者总体；
- 两个前沿候选来自 `src.pipeline.run_all` 的算法演示快照，并转换为人类可读名称；
- 其余候选、趋势业务标签和分群区间用于界面流程演示，不代表名创优品真实数据；
- 页面中的人工决策只写入浏览器会话，不触发采购、打样、定价或上架。
